# vastai_endpoint (Resource)

Manages a Vast.ai serverless endpoint. Endpoints provide autoscaling inference serving with configurable load balancing, cold start behavior, and worker scaling. Worker groups are bound to endpoints via the `vastai_worker_group` resource.

## Example Usage

```terraform
# Create a serverless inference endpoint with autoscaling
resource "vastai_endpoint" "llm_serving" {
  endpoint_name = "llm-inference-v1"

  # Autoscaling configuration
  min_load     = 0
  target_util  = 0.85
  cold_mult    = 2.0
  cold_workers = 3
  max_workers  = 10
}
```

## Schema

### Required

- `endpoint_name` (String) Name of the serverless endpoint.

### Optional

- `cold_mult` (Number) Cold capacity as multiple of hot capacity (default: 2.5).
- `cold_workers` (Number) Minimum cold workers when no load (default: 5).
- `endpoint_state` (String) Endpoint runtime state (active, suspended, stopped). New endpoints start as active.
- `max_workers` (Number) Maximum workers the endpoint can have (default: 20).
- `min_cold_load` (Number) Minimum floor load allowing cold workers (default: 0).
- `min_load` (Number) Minimum floor load in perf units/s (default: 0).
- `target_util` (Number) Target capacity utilization fraction (default: 0.9).
- `timeouts` (Block, Optional) (see [below for nested schema](#nestedblock--timeouts))
