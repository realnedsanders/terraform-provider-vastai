# vastai_instance (Resource)

Manages a Vast.ai GPU instance. Instances are created from GPU offers and support full lifecycle management including start/stop, label and bid price updates, template changes, SSH key attachment, and spot instance preemption handling.

Use the `vastai_gpu_offers` data source to find available offers, then reference the offer ID when creating an instance.

## Example Usage

```terraform
# Search for an affordable RTX 4090 offer
data "vastai_gpu_offers" "rtx4090" {
  gpu_name           = "RTX 4090"
  num_gpus           = 1
  max_price_per_hour = 0.50
  limit              = 5
}

# Create an SSH key for secure access
resource "vastai_ssh_key" "my_key" {
  ssh_key = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIExample user@workstation"
}

# Launch a GPU instance from the best offer
resource "vastai_instance" "training" {
  offer_id = data.vastai_gpu_offers.rtx4090.most_affordable.id
  disk_gb  = 50
  image    = "pytorch/pytorch:2.1.0-cuda12.1-cudnn8-runtime"
  label    = "ml-training-run"

  ssh_key_ids = [vastai_ssh_key.my_key.id]

  env = {
    WANDB_API_KEY = "your-wandb-key"
    PROJECT_NAME  = "my-experiment"
  }

  onstart = "cd /workspace && pip install -r requirements.txt"
}
```

## Schema

### Required

- `disk_gb` (Number) Local disk partition size in GB. Cannot be changed after creation.
- `offer_id` (Number) ID of the GPU offer to create this instance from. Obtain from the vastai_gpu_offers data source. Changing this forces a new resource.

### Optional

- `bid_price` (Number) Bid price per GPU per hour in USD. Set to create an interruptible (spot) instance. Omit for on-demand pricing. Can be updated in-place to change bid amount.
- `cancel_unavail` (Boolean) Cancel instance creation if the selected offer becomes unavailable.
- `env` (Map of String) Environment variables as key-value pairs passed to the instance container. Can be updated via template update.
- `image` (String) Docker image to launch (e.g., 'pytorch/pytorch:latest'). Can be set via template. If not specified, the template's image is used.
- `image_login` (String, Sensitive) Docker registry credentials for private image pulls (format: '-u user -p pass registry'). This value is sensitive and will not be displayed in plan output.
- `label` (String) Human-readable label for the instance. Can be updated in-place.
- `onstart` (String) Startup script executed when the instance boots. Can be updated via template update.
- `ssh_key_ids` (Set of String) Set of SSH key IDs to attach to this instance. Keys are attached/detached incrementally when the set changes. Use IDs from vastai_ssh_key resources.
- `status` (String) Desired instance state. Set to 'running' or 'stopped' to start/stop the instance without destroying and recreating it. Defaults to 'running' on creation.
- `template_hash_id` (String) Hash ID of a Vast.ai template to apply to this instance. Can be updated in-place.
- `timeouts` (Block, Optional) (see [below for nested schema](#nestedblock--timeouts))
- `use_jupyter_lab` (Boolean) Enable JupyterLab access to this instance. Defaults to the template setting if not specified.
- `use_ssh` (Boolean) Enable SSH access to this instance. Defaults to the template setting if not specified.

### Read-Only

- `actual_status` (String) Current actual status of the instance as reported by the API (e.g., 'running', 'loading', 'exited').
- `cost_per_hour` (Number) Current cost per hour in USD for this instance.
- `cpu_cores` (Number) Number of effective CPU cores allocated to this instance.
- `cpu_ram_gb` (Number) Total CPU RAM in GB.
- `created_at` (String) Timestamp when the instance was created.
- `geolocation` (String) Geographic location of the host machine.
- `gpu_name` (String) GPU model name (e.g., 'RTX 4090', 'A100 SXM4').
- `gpu_ram_gb` (Number) Total GPU VRAM in GB.
- `id` (String) Unique contract ID of the instance. Used as the primary identifier for all operations.
- `inet_down_mbps` (Number) Internet download speed in Mbps.
- `inet_up_mbps` (Number) Internet upload speed in Mbps.
- `is_bid` (Boolean) Whether this instance uses bid (spot/interruptible) pricing.
- `machine_id` (Number) Physical machine ID hosting this instance.
- `num_gpus` (Number) Number of GPUs allocated to this instance.
- `reliability` (Number) Host machine reliability score (0.0 to 1.0).
- `ssh_host` (String) SSH connection hostname for this instance.
- `ssh_port` (Number) SSH connection port for this instance.
- `status_msg` (String) Current status message from the instance (e.g., progress information during loading).
