# vastai_network_volume_offers (Data Source)

Search for available network volume offers on the Vast.ai marketplace. Returns a filtered list of offers with a convenience `most_affordable` attribute for the cheapest matching offer.

## Example Usage

```terraform
# Search for network volume offers
data "vastai_network_volume_offers" "available" {
  limit = 10
}

output "cheapest_nv_offer" {
  value = data.vastai_network_volume_offers.available.most_affordable.id
}
```

## Schema

### Optional

- `allocated_storage` (Number) Storage amount in GB for pricing calculations. Affects the computed storage_cost in results. Defaults to 1.0 if not set. Set this to match your intended volume size for accurate pricing.
- `disk_bw` (Number) Minimum disk bandwidth in MB/s.
- `disk_space` (Number) Minimum disk space in GB. Only offers with at least this much disk space will be returned.
- `geolocation` (String) Filter by geographic location (e.g., 'US', 'EU'). Matches against the offer's geolocation field.
- `inet_down` (Number) Minimum internet download speed in Mbps.
- `inet_up` (Number) Minimum internet upload speed in Mbps.
- `limit` (Number) Maximum number of offers to return. Valid range: 1 to 1000. Default: 10.
- `max_storage_cost` (Number) Maximum storage cost per GB per month in USD. Only offers at or below this cost will be returned.
- `order_by` (String) Field to sort results by. Valid values: 'storage_cost', 'disk_space', 'inet_up', 'inet_down', 'reliability', 'duration'. Default: 'storage_cost'.
- `raw_query` (String) Raw query JSON string to pass directly to the Vast.ai network volume search API. When set, all structured filter attributes are ignored. Use this for advanced queries not supported by the structured filters.
- `reliability` (Number) Minimum host reliability score. Valid range: 0.0 to 1.0.
- `static_ip` (Boolean) If true, only return offers from machines with static IP addresses.
- `verified` (Boolean) If true, only return offers from verified machines.

### Read-Only

- `most_affordable` (Attributes) The most affordable offer from the results (first result when sorted by storage_cost). Convenience attribute to avoid indexing into the offers list. (see [below for nested schema](#nestedatt--most_affordable))
- `offers` (Attributes List) List of network volume offers matching the search criteria, sorted by the `order_by` field. (see [below for nested schema](#nestedatt--offers))
