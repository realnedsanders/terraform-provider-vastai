    Create the vastai_api_key and vastai_environment_variable resources following the SSH key resource pattern exactly.

    **apikey/models.go:**
    - `ApiKeyResourceModel` struct: `ID types.String`, `Name types.String`, `Key types.String` (the actual secret key value), `Permissions types.String` (JSON string), `CreatedAt types.String`, `Timeouts timeouts.Value`

    **apikey/resource_api_key.go** (ACCT-01):
    - `ApiKeyResource` struct with `client *client.VastAIClient`
    - Interface assertions: `resource.Resource`, `resource.ResourceWithImportState`
    - TypeName: `vastai_api_key`
    - Schema attributes:
      - `id`: StringAttribute, Computed, UseStateForUnknown (stable after create)
      - `name`: StringAttribute, Required, ForceNew (immutable). Validator: stringvalidator.LengthAtLeast(1)
      - `key`: StringAttribute, Computed, Sensitive: true, UseStateForUnknown. Description: "The API key value. Only available on creation, never returned by subsequent reads."
        This is the critical Pitfall 1 handling: key is set once on create and preserved via UseStateForUnknown.
      - `permissions`: StringAttribute, Required, ForceNew (immutable since no update endpoint). Description: "JSON string of permission configuration."
        Use a custom validator that checks valid JSON (json.Valid).
      - `created_at`: StringAttribute, Computed, UseStateForUnknown
      - `timeouts`: timeouts block (create, read, delete)
    - Create: Call client.ApiKeys.Create(name, permissions). Store returned key value AND id in state. Convert permissions json.RawMessage to string for state.
    - Read: Call client.ApiKeys.List(), find by ID. Update name, permissions, created_at from API. Do NOT touch the `key` field (UseStateForUnknown preserves it).
    - Update: Return error diagnostic "API keys are immutable. Modify requires replacement." (ForceNew on name/permissions handles this at plan time.)
    - Delete: Call client.ApiKeys.Delete(id).
    - ImportState: Import by ID string, set ID attribute, let Read populate the rest. Key field will be empty on import (document this: "The key value cannot be recovered after creation").

    **envvar/models.go:**
    - `EnvVarResourceModel` struct: `ID types.String` (stores the key/name), `Key types.String`, `Value types.String`, `Timeouts timeouts.Value`

    **envvar/resource_env_var.go** (ACCT-02):
    - `EnvVarResource` struct with `client *client.VastAIClient`
    - Interface assertions: `resource.Resource`, `resource.ResourceWithImportState`
    - TypeName: `vastai_environment_variable`
    - Schema attributes:
      - `id`: StringAttribute, Computed, UseStateForUnknown. Description: "The environment variable name (same as key)."
      - `key`: StringAttribute, Required, ForceNew. Validator: stringvalidator.LengthAtLeast(1). Description: "Environment variable name."
      - `value`: StringAttribute, Required, Sensitive: true. Description: "Environment variable value."
      - `timeouts`: timeouts block
    - Create: Call client.EnvVars.Create(key, value). Set id = key in state.
    - Read: Call client.EnvVars.List(), find entry matching key. If not found, remove from state (resp.State.RemoveResource). If found, update value in state.
      Note: The value IS returned by the List API, so we can detect drift.
    - Update: Call client.EnvVars.Update(key, value).
    - Delete: Call client.EnvVars.Delete(key).
    - ImportState: Import by key name (the name IS the ID). Set both id and key attributes.

    For both test files: follow ssh_key_test.go pattern using httptest mock server. Test Create, Read (via list), Delete, and ImportState. For API key, additionally test that the key field is preserved across reads (UseStateForUnknown behavior).

    **subaccount/resource_subaccount.go** (ACCT-06):
    - `SubaccountResource` struct with `client *client.VastAIClient`
    - Interface assertions: `resource.Resource`, `resource.ResourceWithImportState`
    - TypeName: `vastai_subaccount`
    - Schema attributes:
      - `id`: StringAttribute, Computed, UseStateForUnknown
      - `email`: StringAttribute, Required, ForceNew. Validator: stringvalidator.LengthAtLeast(1)
      - `username`: StringAttribute, Required, ForceNew. Validator: stringvalidator.LengthAtLeast(1)
      - `password`: StringAttribute, Required, Sensitive: true, ForceNew. Description: "Password for the subaccount. Write-only, never returned by API."
      - `host_only`: BoolAttribute, Optional, Computed, ForceNew. Default: false. Description: "Whether this is a host-only subaccount."
      - `timeouts`: timeouts block
    - Description: "Manages a Vast.ai subaccount. NOTE: Subaccounts cannot be deleted via the API. Destroying this resource removes it from Terraform state only."
    - Create: Call client.Subaccounts.Create(email, username, password, hostOnly). Store returned ID.
    - Read: Call client.Subaccounts.List(), find by ID. Update email, username from API. Password is write-only (UseStateForUnknown).
    - Update: Return error "Subaccounts are immutable after creation." (ForceNew on all fields handles this.)
    - Delete: **NO API CALL.** Add a warning diagnostic: `resp.Diagnostics.AddWarning("Subaccount Not Deleted", "The subaccount was removed from Terraform state but was NOT deleted from Vast.ai. Subaccount deletion is not supported by the Vast.ai API.")`. Then simply return (state automatically removed).
    - ImportState: Import by ID. Password will be empty on import (write-only).

    Test file: Test Create (mock POST returning ID), Read via List (mock GET returning list with matching ID), Delete (verify NO HTTP request is made -- the mock server should NOT receive any delete request). Test import state.
