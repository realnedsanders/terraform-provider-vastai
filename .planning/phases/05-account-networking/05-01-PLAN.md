  <action>
    Create 4 client service files following the exact pattern from ssh_keys.go and endpoints.go.
    Each service file has: typed API model structs, a Service struct with `client *VastAIClient`, and methods for each API operation.
    Each test file has: httptest mock server, tests for each operation covering success and error paths.

    **api_keys.go** (ACCT-01):
    - `ApiKeyService` struct with `client *VastAIClient`
    - `ApiKey` struct: `ID int`, `Name string`, `Key string` (json omitempty -- only on create), `Permissions json.RawMessage`, `KeyParams string` (omitempty), `CreatedAt string` (omitempty)
    - `Create(ctx, name string, permissions json.RawMessage) (*ApiKey, error)` -- POST `/auth/apikeys/` with `{"name": name, "permissions": perms}`. Returns full ApiKey including key value.
    - `List(ctx) ([]ApiKey, error)` -- GET `/auth/apikeys/`. Returns array of ApiKey (without key value).
    - `Delete(ctx, id int) error` -- DELETE `/auth/apikeys/{id}/`. Standard path-based delete.
    - No Update method -- API keys are immutable after creation per research.

    **env_vars.go** (ACCT-02):
    - `EnvVarService` struct with `client *VastAIClient`
    - `EnvVarMap` struct: `Secrets map[string]string` (json:"secrets")
    - `Create(ctx, key, value string) error` -- POST `/secrets/` with `{"key": key, "value": value}`
    - `List(ctx) (map[string]string, error)` -- GET `/secrets/` returns EnvVarMap, extract .Secrets
    - `Update(ctx, key, value string) error` -- PUT `/secrets/` with `{"key": key, "value": value}`
    - `Delete(ctx, key string) error` -- DELETE `/secrets/` using DeleteWithBody with `{"key": key}` (Pitfall 2)

    **teams.go** (ACCT-03, ACCT-04, ACCT-05):
    Combined service for team, roles, and members since they share the `/team/` API namespace.
    - `TeamService` struct with `client *VastAIClient`
    - `Team` struct: `ID int`, `TeamName string` (json:"team_name"), plus any other returned fields
    - `TeamRole` struct: `ID int`, `Name string`, `Permissions json.RawMessage`
      NOTE on permissions: D-02 originally specified a string set, but research revealed the API uses nested JSON objects (e.g., `{"api":{"instance_read":{...}}}`), not a flat set. Storing as json.RawMessage is the correct approach (D-02 revised -- API requires nested JSON, not flat string set).
    - `TeamMember` struct: `ID int`, `Email string`, `Role string`, plus other returned fields
    - Team CRUD:
      - `CreateTeam(ctx, teamName string) (*Team, error)` -- POST `/team/` with `{"team_name": name}`
      - `DestroyTeam(ctx) error` -- DELETE `/team/` (no params, parameterless delete per research)
    - Role CRUD:
      - `CreateRole(ctx, name string, permissions json.RawMessage) (*TeamRole, error)` -- POST `/team/roles/` with `{"name": name, "permissions": perms}`
      - `ListRoles(ctx) ([]TeamRole, error)` -- GET `/team/roles-full/`
      - `GetRole(ctx, name string) (*TeamRole, error)` -- GET `/team/roles/{name}/`
      - `UpdateRole(ctx, id int, name string, permissions json.RawMessage) (*TeamRole, error)` -- PUT `/team/roles/{id}/` (Pitfall 3: update uses ID, not name)
      - `DeleteRole(ctx, name string) error` -- DELETE `/team/roles/{name}/` (Pitfall 3: delete uses name)
    - Member operations:
      - `InviteMember(ctx, email, role string) error` -- POST `/team/invite/?email={email}&role={role}` (Pitfall 5: query params, NOT JSON body). Encode email/role directly in URL path.
      - `ListMembers(ctx) ([]TeamMember, error)` -- GET `/team/members/`
      - `RemoveMember(ctx, id int) error` -- DELETE `/team/members/{id}/`

    **subaccounts.go** (ACCT-06):
    - `SubaccountService` struct with `client *VastAIClient`
    - `Subaccount` struct: `ID int`, `Email string`, `Username string`
    - `SubaccountCreateRequest` struct: `Email string`, `Username string`, `Password string`, `HostOnly bool` (json:"host_only"), `ParentID string` (json:"parent_id")
    - `SubaccountListResponse` struct: `Users []Subaccount` (json:"users")
    - `Create(ctx, email, username, password string, hostOnly bool) (*Subaccount, error)` -- POST `/users/` with parent_id="me". Parse response for ID.
    - `List(ctx) ([]Subaccount, error)` -- GET `/subaccounts?owner=me`. Unwrap .Users from response.
    - No Delete or Update methods -- API has no delete endpoint per research (Pitfall 8).

    For all test files: use httptest.NewServer pattern from ssh_keys_test.go. Test success and error (non-200 status) for each operation.
