    **Create `internal/services/endpoint/resource_endpoint.go`:**

    Package `endpoint`. Follow exact pattern from `resource_network_volume.go`.

    Interface implementations:
    - `var _ resource.Resource = &EndpointResource{}`
    - `var _ resource.ResourceWithConfigure = &EndpointResource{}`
    - `var _ resource.ResourceWithImportState = &EndpointResource{}`

    Type name: `vastai_endpoint` (via req.ProviderTypeName + "_endpoint")

    Schema attributes (per D-04 and D-05):
    - `id`: StringAttribute, Computed, UseStateForUnknown
    - `endpoint_name`: StringAttribute, Required, description "Name of the serverless endpoint"
    - `min_load`: Float64Attribute, Optional+Computed, validator float64validator.AtLeast(0), description "Minimum floor load in perf units/s (default: 0)"
    - `min_cold_load`: Float64Attribute, Optional+Computed, validator float64validator.AtLeast(0), description "Minimum floor load allowing cold workers (default: 0)"
    - `target_util`: Float64Attribute, Optional+Computed, validator float64validator.Between(0, 1), description "Target capacity utilization fraction (default: 0.9)"
    - `cold_mult`: Float64Attribute, Optional+Computed, validator float64validator.AtLeast(1.0), description "Cold capacity as multiple of hot capacity (default: 2.5)"
    - `cold_workers`: Int64Attribute, Optional+Computed, validator int64validator.AtLeast(0), description "Minimum cold workers when no load (default: 5)"
    - `max_workers`: Int64Attribute, Optional+Computed, validator int64validator.AtLeast(0), description "Maximum workers the endpoint can have (default: 20)"
    - `endpoint_state`: StringAttribute, Optional+Computed, validator stringvalidator.OneOf("active", "suspended", "stopped"), description "Endpoint runtime state (active, suspended, stopped). New endpoints start as active."
    - `timeouts`: timeouts block with create (default 5m), read (default 2m), update (default 5m), delete (default 5m)

    CRUD methods:
    - **Create**: Read plan into EndpointResourceModel, build CreateEndpointRequest from model fields (use .ValueFloat64()/.ValueInt64()), call r.client.Endpoints.Create(ctx, req). Map returned Endpoint to model (readEndpointIntoModel helper). Save state. If endpoint_state is set and not "active", issue an Update call after create to set state (per Pitfall 6).
    - **Read**: Get ID from state, parse to int, call r.client.Endpoints.List(ctx), find by ID. If not found, resp.State.RemoveResource (remove from state). If found, call readEndpointIntoModel to map API response to TF model.
    - **Update**: Read plan into model, build UpdateEndpointRequest from model (use pointer fields for optional values). Call r.client.Endpoints.Update(ctx, id, req). Then re-read via List to get updated state (read-after-write).
    - **Delete**: Get ID from state, call r.client.Endpoints.Delete(ctx, id).
    - **ImportState**: resource.ImportStatePassthroughID on path "id".

    Helper `readEndpointIntoModel(endpoint *client.Endpoint, model *EndpointResourceModel)`:
    - model.ID = types.StringValue(strconv.Itoa(endpoint.ID))
    - model.EndpointName = types.StringValue(endpoint.EndpointName)
    - model.MinLoad = types.Float64Value(endpoint.MinLoad)
    - model.MinColdLoad = types.Float64Value(endpoint.MinColdLoad)
    - model.TargetUtil = types.Float64Value(endpoint.TargetUtil)
    - model.ColdMult = types.Float64Value(endpoint.ColdMult)
    - model.ColdWorkers = types.Int64Value(int64(endpoint.ColdWorkers))
    - model.MaxWorkers = types.Int64Value(int64(endpoint.MaxWorkers))
    - model.EndpointState = types.StringValue(endpoint.EndpointState)
