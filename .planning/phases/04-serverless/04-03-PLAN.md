    - LaunchArgs types.String `tfsdk:"launch_args"` -- Optional
    - GpuRAM types.Float64 `tfsdk:"gpu_ram"` -- Optional (estimated GPU RAM requirement)
    - TestWorkers types.Int64 `tfsdk:"test_workers"` -- Optional+Computed (default 3)
    - ColdWorkers types.Int64 `tfsdk:"cold_workers"` -- Optional
    - Timeouts timeouts.Value `tfsdk:"timeouts"`

    Note: Per Pitfall 3 in research, min_load/target_util/cold_mult are "not currently used at the workergroup level". OMIT these from the worker group schema to avoid confusion. The API accepts them but they have no effect -- autoscaling is driven by the endpoint. If users set these, they'd be confused when the endpoint values take effect. Document in the resource description that autoscaling is configured on the endpoint.

    **Create `internal/services/workergroup/resource_worker_group.go`:**

    Package `workergroup`. Follow exact pattern from `resource_endpoint.go`.

    Interface implementations:
    - `var _ resource.Resource = &WorkerGroupResource{}`
    - `var _ resource.ResourceWithConfigure = &WorkerGroupResource{}`

    Type name: `vastai_worker_group` (via req.ProviderTypeName + "_worker_group")

    Schema description: "Manages a Vast.ai worker group. Worker groups bind to serverless endpoints and define the compute configuration (template, GPU requirements, search parameters). Autoscaling behavior is controlled at the endpoint level via the vastai_endpoint resource."

    Schema attributes:
    - `id`: StringAttribute, Computed, UseStateForUnknown
    - `endpoint_id`: Int64Attribute, Required, RequiresReplace plan modifier (changing endpoint requires recreate per D-02), description "ID of the parent endpoint this worker group belongs to. Changing this forces a new resource."
    - `endpoint_name`: StringAttribute, Optional+Computed, description "Name of the parent endpoint (computed from endpoint_id if not set)"
    - `template_hash`: StringAttribute, Optional, AtLeastOneOf with path.MatchRoot("template_id"), description "Template hash for worker instances. Either template_hash or template_id must be provided."
    - `template_id`: Int64Attribute, Optional, AtLeastOneOf with path.MatchRoot("template_hash"), description "Numeric template ID for worker instances. Either template_hash or template_id must be provided."
    - `search_params`: StringAttribute, Optional, description "Offer search filter string (e.g. 'gpu_ram>=23 num_gpus=2 gpu_name=RTX_4090'). The API may apply default filters (verified=True rentable=True rented=False)."
    - `launch_args`: StringAttribute, Optional, description "Instance launch arguments string"
    - `gpu_ram`: Float64Attribute, Optional, validator float64validator.AtLeast(0), description "Estimated GPU RAM requirement in GB"
    - `test_workers`: Int64Attribute, Optional+Computed, validator int64validator.AtLeast(0), description "Number of workers for initial performance estimate (default: 3)"
    - `cold_workers`: Int64Attribute, Optional, validator int64validator.AtLeast(0), description "Minimum cold workers for this worker group"
    - `timeouts`: timeouts block with create (default 5m), read (default 2m), update (default 5m), delete (default 5m)

    CRUD methods:
    - **Create**: Read plan into WorkerGroupResourceModel, build CreateWorkerGroupRequest. Set endpoint_id from model. Set template_hash and/or template_id from model. Set search_params, launch_args, gpu_ram if not null. For API-required autoscaling fields (per research: min_load, target_util, cold_mult are sent in create body), send sensible defaults: MinLoad=0, TargetUtil=0.9, ColdMult=2.0. Set TestWorkers from model (default 3 if null), ColdWorkers from model. Call r.client.WorkerGroups.Create(ctx, req). Map returned WorkerGroup to model via readWorkerGroupIntoModel helper. Save state.
    - **Read**: Get ID from state, parse to int, call r.client.WorkerGroups.List(ctx), find by ID. If not found, resp.State.RemoveResource. If found, readWorkerGroupIntoModel.
    - **Update**: Read plan into model, build UpdateWorkerGroupRequest from model. Set AutoJobID=id. Include non-null fields. Call r.client.WorkerGroups.Update(ctx, id, req). Re-read via List (read-after-write pattern).
    - **Delete**: Get ID from state, call r.client.WorkerGroups.Delete(ctx, id).
    - **ImportState**: resource.ImportStatePassthroughID on path "id".

    Helper `readWorkerGroupIntoModel(wg *client.WorkerGroup, model *WorkerGroupResourceModel)`:

    - model.SearchParams = types.StringValue(wg.SearchParams) -- if empty, set types.StringNull()
    - model.LaunchArgs = types.StringValue(wg.LaunchArgs) -- if empty, set types.StringNull()
    - model.GpuRAM = types.Float64Value(wg.GpuRAM) -- if 0, set types.Float64Null()
