NetworkVolumeOffersDataSourceModel -- same as VolumeOffersDataSourceModel but results use NetworkVolumeOfferModel.

NetworkVolumeOfferModel -- includes all VolumeOfferModel fields PLUS network-volume-specific fields:
- `cluster_id`: Int64
- `nw_disk_min_bw`: Float64 (description: "Minimum network disk bandwidth in MB/s")
- `nw_disk_max_bw`: Float64 (description: "Maximum network disk bandwidth in MB/s")
- `nw_disk_avg_bw`: Float64 (description: "Average network disk bandwidth in MB/s")

Note: NetworkVolumeOfferModel does NOT have cuda_max_good, cpu_ghz, disk_bw, disk_name, driver_version, machine_id that local volume offers have. Use the exact response fields from research: id, disk_space, storage_cost, inet_up, inet_down, reliability, duration, verification, host_id, cluster_id, geolocation, nw_disk_min_bw, nw_disk_max_bw, nw_disk_avg_bw.
