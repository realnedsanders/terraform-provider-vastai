**internal/services/volume/resource_volume.go** -- vastai_volume resource:

Follow template/resource_template.go structure exactly:

1. Interface compliance vars: resource.Resource, resource.ResourceWithConfigure, resource.ResourceWithImportState
2. VolumeResource struct with client *client.VastAIClient
3. NewVolumeResource() factory function
4. Metadata: type name = "vastai_volume"
5. Schema with these attributes:
   - `id`: StringAttribute, Computed, UseStateForUnknown (volume contract ID as string for TF compatibility)
   - `offer_id`: Int64Attribute, Required, RequiresReplace. Description: "ID of the volume offer to create from (from vastai_volume_offers data source)." Validator: int64validator.AtLeast(1)
   - `size`: Int64Attribute, Required, RequiresReplace. Description: "Volume size in GB." Validator: int64validator.AtLeast(1). Pitfall 7: int for create, not float.
   - `name`: StringAttribute, Optional. Description: "Optional name/label for the volume."
   - `clone_from_id`: Int64Attribute, Optional, RequiresReplace. Description: "Source volume ID to clone from. When set, creates by cloning (POST /volumes/copy/) instead of creating from offer. Per D-01." Validator: int64validator.AtLeast(1)
   - `disable_compression`: BoolAttribute, Optional. Description: "Disable compression during clone. Only used when clone_from_id is set."
   - `status`: StringAttribute, Computed, UseStateForUnknown
   - `disk_space`: Float64Attribute, Computed. Description: "Actual allocated disk space in GB."
   - `machine_id`: Int64Attribute, Computed, UseStateForUnknown
   - `geolocation`: StringAttribute, Computed, UseStateForUnknown
   - `inet_up`: Float64Attribute, Computed
   - `inet_down`: Float64Attribute, Computed
   - `reliability`: Float64Attribute, Computed
   - `disk_name`: StringAttribute, Computed, UseStateForUnknown
   - `driver_version`: StringAttribute, Computed
   - `host_id`: Int64Attribute, Computed, UseStateForUnknown
   - `verification`: StringAttribute, Computed
   - Timeouts block: Create (5min default), Read (5min), Delete (5min). Clone gets 30 minutes -- use Create timeout since clone is a create variant.

6. Configure: Extract *client.VastAIClient from req.ProviderData (same pattern as template resource)

7. Create:
   - Read timeout from config
   - If clone_from_id is set: use client.Volumes.Clone() with SourceID=clone_from_id, DestOfferID=offer_id, Size=size (as float64), DisableCompression. Then read back via List("local_volume") to get the new volume. Per D-01 and Pitfall 4 (clone needs both source AND dest offer ID).
   - If clone_from_id is NOT set: use client.Volumes.Create() with Size=size (as int), OfferID=offer_id, Name=name.
   - After create/clone, call readVolumeIntoModel() to populate all computed fields.
   - Set the ID in state (convert int to string via strconv.FormatInt).

8. Read:
   - Parse ID from state (string to int via strconv.ParseInt)
   - Call client.Volumes.List("local_volume")
   - Filter by ID (same read-via-list pattern as SSH keys per Pitfall 3)
   - If not found, call resp.State.RemoveResource() to remove from state
   - Map all API fields to model: Status, DiskSpace, MachineID, Geolocation, InetUp, InetDown, Reliability, DiskName, DriverVersion, HostID, Verification

9. Update: Return error diagnostic "volumes are immutable; changes require replacement". All mutable attributes are ForceNew so this should never be called, but implement defensively.

10. Delete:
    - Parse ID from state
    - Call client.Volumes.Delete(id)
    - Log successful deletion

11. ImportState: passthrough.ImportStateIdContext for "id" attribute. The user provides the volume contract ID.

NOTE on list/unlist: Per research, marketplace list/unlist are HOST-only operations (Pitfall 1). This provider targets tenants. OMIT list/unlist entirely. This is a scope adjustment from the STOR-01 requirement text -- document in summary.
