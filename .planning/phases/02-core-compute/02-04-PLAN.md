    **Implement CRUD methods in resource_instance.go, replacing the stubs from Task 1:**

    **Create method:**
    1. Read plan into InstanceResourceModel
    2. Get create timeout via `model.Timeouts.Create(ctx, 10*time.Minute)` per D-10
    3. Build `CreateInstanceRequest` from model:
       - `ClientID: "me"`
       - `Image` from model (or empty if using template)
       - `Disk` from model.DiskGB
       - `Price` as *float64 pointer -- nil if BidPrice is null (on-demand), else &value (spot/bid)
       - `Label`, `Onstart`, `TemplateHashID` from model
       - Build `Runtype` string from use_ssh/use_jupyter_lab bools per Pitfall 7: if use_ssh=true, include "ssh_direc ssh_proxy"; if use_jupyter_lab=true, include "jupyter_direc"
       - Convert `Env` map to the API format: map[string]string where keys are env var names and values are their values. Port mappings should be expressed as "-p PORT:PORT": "1" entries. (per Pitfall 3)
       - `ImageLogin`, `CancelUnavail`, `UseJupyterLab` from model
    4. Call `client.Instances.Create(ctx, offerID, req)`
    5. Handle 404/400 from creation: error with "Offer {id} is no longer available. Run `terraform plan` again to search for current offers." per D-06/Pitfall 2
    6. Call `client.Instances.WaitForStatus(ctx, resp.NewContract, "running", createTimeout)` per COMP-05
    7. Map the returned Instance to model (convert types, divide RAM by 1000 for GB per Pitfall 6)
    8. If `ssh_key_ids` is set, attach each key: for each ID in the set, look up the public key content (Read from SSH keys list), then call `client.SSHKeys.AttachToInstance(ctx, instanceID, publicKey)` per COMP-08. NOTE: The attach endpoint takes the full SSH key content, not a key ID. So the resource must resolve ID to content.
    9. Save state

    **Read method:**
    1. Get state into model
    2. Parse ID: `strconv.Atoi(model.ID.ValueString())`
    3. Call `client.Instances.Get(ctx, id)`
    4. Handle 404: remove from state (resp.State.RemoveResource) -- instance was destroyed externally
    5. **Preemption detection per D-09:**
       - If `instance.IsBid == true` (spot instance) AND `instance.IntendedStatus == "running"` AND `instance.ActualStatus` is one of ["stopped", "offline", "exited"]:
         - This is preemption. Log at WARN: "Instance {id} appears to have been preempted (is_bid=true, intended=running, actual={status})."
         - Remove from state silently: `resp.State.RemoveResource(ctx)`
         - Return early
       - If NOT the above condition (on-demand instance, or intended_status was already "stopped"):
         - This is a normal state change. Update model from API response normally. Do NOT remove from state.
    6. Map Instance to model (convert types, RAM/1000 for GB)
    7. Save state

    **Update method:**
    1. Read plan and state into separate models
    2. Parse ID
    3. Detect what changed and apply updates in order:
       a. **Status changed (start/stop per COMP-02):** If plan.Status != state.Status:
          - If plan.Status == "running": `client.Instances.Start(ctx, id)`, then WaitForStatus "running"
          - If plan.Status == "stopped": `client.Instances.Stop(ctx, id)`, then WaitForStatus "stopped"
       b. **Label changed (per COMP-03):** `client.Instances.SetLabel(ctx, id, newLabel)`
       c. **Bid price changed (per COMP-03, D-11):** `client.Instances.ChangeBid(ctx, id, newPrice)`
       d. **Template/image/env/onstart changed (per COMP-03):** Build `UpdateTemplateRequest`, call `client.Instances.UpdateTemplate(ctx, id, req)`
       e. **SSH key IDs changed (per COMP-08):**
          - Compute added = plan.SSHKeyIDs - state.SSHKeyIDs
          - Compute removed = state.SSHKeyIDs - plan.SSHKeyIDs
          - For each added: resolve ID to public key, call AttachToInstance
          - For each removed: call DetachFromInstance with key ID
    4. Re-read instance state: `client.Instances.Get(ctx, id)`
    5. Map to model, save state

    **Delete method:**
    1. Get delete timeout
    2. Parse ID
    3. Call `client.Instances.Destroy(ctx, id)`
    4. Optionally wait for 404 (instance gone) with short timeout

    **ImportState method per IMPT-01/IMPT-02:**
    `resource.ImportStatePassthroughID(ctx, path.Root("id"), req, resp)`

    **Helper function: mapInstanceToModel(instance *client.Instance, model *InstanceResourceModel)**
    Converts API Instance struct to Terraform model:
    - `model.ID = types.StringValue(strconv.Itoa(instance.ID))`
    - `model.MachineID = types.Int64Value(int64(instance.MachineID))`
    - `model.ActualStatus = types.StringValue(instance.ActualStatus)`
    - `model.Status = types.StringValue(instance.IntendedStatus)` (status represents desired state)
    - GPU/CPU RAM: divide by 1000 for GB per Pitfall 6
    - `model.IsBid = types.BoolValue(instance.IsBid)`
    - etc.
