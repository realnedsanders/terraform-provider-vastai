    - `vastai_team_member` — invite with email and role

    - `vastai_overlay` — create bound to cluster
    - `vastai_overlay_member` — join instance to overlay
    - `vastai_subaccount` — create with username

    - `vastai_gpu_offers` — search with gpu_name="RTX_4090", num_gpus=1, limit
    - `vastai_instance` — lookup by id
    - `vastai_instances` — list with optional label filter
    - `vastai_templates` — search by query string
    - `vastai_ssh_keys` — list all keys
    - `vastai_endpoints` — list all endpoints
    - `vastai_volume_offers` — search with filters
    - `vastai_network_volume_offers` — search with filters
    - `vastai_user` — read current account profile
    - `vastai_invoices` — list billing history
    - `vastai_audit_logs` — list account activity

    **Step 3: Create tfplugindocs template files.** Each template follows this pattern for resources:
    ```
    ---
    page_title: "vastai_{name} Resource - terraform-provider-vastai"
    subcategory: "{Category}"
    description: |-
      {One-line description}
    ---

    # vastai_{name} (Resource)

    {2-3 sentence description of what the resource manages and key behaviors}

    ## Example Usage

    {{ tffile "examples/resources/vastai_{name}/resource.tf" }}

    {{ .SchemaMarkdown | trimspace }}

    ## Import

    {name} can be imported using the resource ID:

    ```shell
    terraform import vastai_{name}.example {id}
    ```
    ```

    For data sources, same pattern but no Import section:
    ```
    ---
    page_title: "vastai_{name} Data Source - terraform-provider-vastai"
    subcategory: "{Category}"
    description: |-
      {One-line description}
    ---

    # vastai_{name} (Data Source)

    {2-3 sentence description}

    ## Example Usage

    {{ tffile "examples/data-sources/vastai_{name}/data-source.tf" }}

    {{ .SchemaMarkdown | trimspace }}
    ```

    Use these subcategories:
    - "Compute" for: instance, template, ssh_key, gpu_offers, templates, ssh_keys
    - "Storage" for: volume, network_volume, volume_offers, network_volume_offers
    - "Serverless" for: endpoint, worker_group, endpoints
    - "Account" for: api_key, environment_variable, subaccount, user, invoices, audit_logs
    - "Team" for: team, team_role, team_member
    - "Networking" for: cluster, cluster_member, overlay, overlay_member

    For resources with composite IDs (cluster_member, overlay_member), note the import format uses `cluster_id/machine_id` or `overlay_id/instance_id`.

    For resources that are create-only or have no-op destroy (subaccount, overlay_member), note this in the description.
