    **Step 3: Create per-resource sweeper files.** Each sweeper file follows this pattern (using instance as example):

    ```go
    package instance

    import (
        "context"
        "fmt"
        "log"
        "strings"

        "github.com/hashicorp/terraform-plugin-testing/helper/resource"
        "github.com/realnedsanders/terraform-provider-vastai/internal/acctest"
    )

    const testResourcePrefix = "tfacc-"

    func init() {
        resource.AddTestSweepers("vastai_instance", &resource.Sweeper{
            Name: "vastai_instance",
            F:    sweepInstances,
        })
    }

    func sweepInstances(region string) error {
        client, err := acctest.SharedSweepClient()
        if err != nil {
            return fmt.Errorf("error getting client: %s", err)
        }

        ctx := context.Background()
        instances, err := client.Instances.List(ctx)
        if err != nil {
            return fmt.Errorf("error listing instances: %s", err)
        }

        for _, inst := range instances {
            if strings.HasPrefix(inst.Label, testResourcePrefix) {
                log.Printf("[INFO] Destroying instance %d (%s)", inst.ID, inst.Label)
                if err := client.Instances.Destroy(ctx, inst.ID); err != nil {
                    log.Printf("[ERROR] Failed to destroy instance %d: %s", inst.ID, err)
                }
            }
        }
        return nil
    }
    ```

    Create sweepers for these 13 resource types (skip read-only cluster_member, overlay_member, team_member, team_role as they are managed via parent resources):

    1. **vastai_instance** — List instances, delete those with label starting with `tfacc-`. Uses `client.Instances.List()` and `client.Instances.Destroy(ctx, id)`. Filter on `inst.Label`.

    2. **vastai_template** — List templates via the data source's search. Templates have hash_id for deletion. Use `client.Templates.Delete(ctx, hashID)`. Filter on template Name starting with `tfacc-`. NOTE: Read the TemplateService to understand how to list templates. If there's a Search/List method, use it. If not, templates may only be searchable. Read the actual client method signatures.

    3. **vastai_ssh_key** — List SSH keys via `client.SSHKeys.List()`, delete those with name starting with `tfacc-`. Uses `client.SSHKeys.Delete(ctx, id)`.

    4. **vastai_volume** — List volumes via `client.Volumes.List(ctx, "")`, delete those with name/label starting with `tfacc-`. Uses `client.Volumes.Delete(ctx, id)`.

    5. **vastai_network_volume** — List via `client.NetworkVolumes.List()`, delete those with name starting with `tfacc-`. Uses `client.NetworkVolumes.Delete(ctx, id)`.

    6. **vastai_endpoint** — List via `client.Endpoints.List()`, delete those with name starting with `tfacc-`. Uses `client.Endpoints.Delete(ctx, id)`. Add dependency: sweeps worker groups first.

    7. **vastai_worker_group** — List via `client.WorkerGroups.List()`, delete those associated with `tfacc-` endpoints. Uses `client.WorkerGroups.Delete(ctx, id)`.

    8. **vastai_api_key** — List via `client.ApiKeys.List()`, delete those with name starting with `tfacc-`. Uses `client.ApiKeys.Delete(ctx, id)`.

    9. **vastai_environment_variable** — List via `client.EnvVars.List()`, delete those with key starting with `tfacc-`. Uses `client.EnvVars.Delete(ctx, key)`.

    10. **vastai_team** — Sweep by destroying via `client.Teams.DestroyTeam()`. NOTE: Team service may not support listing multiple teams. Read the actual client to understand. If teams are account-scoped (one team), skip sweeper or make it conditional.

    11. **vastai_cluster** — List via `client.Clusters.List()`, delete those with name starting with `tfacc-`. Uses `client.Clusters.Delete(ctx, id)`.

    12. **vastai_overlay** — List via `client.Overlays.List()`, delete those with name starting with `tfacc-`. Uses `client.Overlays.Delete(ctx, id)`. Add dependency: sweeps after cluster.

    13. **vastai_subaccount** — List via `client.Subaccounts.List()`. Subaccounts may be create-only with no delete. Read the actual client. If no delete exists, skip sweeper and document why.

    **IMPORTANT:** Before writing each sweeper, read the corresponding client service file to verify:
    - The exact method signatures for List and Delete
    - What fields exist on the response structs (Name, Label, etc.) for prefix matching
    - Whether Delete takes an int ID, string ID, or other parameter

    Register sweeper dependencies where applicable:
    - worker_group sweeper depends on endpoint sweeper (worker groups belong to endpoints)
    - overlay sweeper depends on cluster sweeper (overlays belong to clusters)

    ```makefile
    sweep:
    	@echo "WARNING: This will destroy resources with 'tfacc-' prefix in your Vast.ai account"
    	go test ./internal/services/... -v -sweep=all -timeout 15m
    ```

    Also add `sweep` to the `.PHONY` line.

    **Step 2: Update CI workflow** at `.github/workflows/test.yml`. Add a `sweep` job that runs AFTER the `acceptance` job to clean up any leaked resources:

    ```yaml
    sweep:
      name: Sweep Test Resources
      if: github.ref == 'refs/heads/main'
      runs-on: ubuntu-latest
      timeout-minutes: 15
      needs: acceptance
      if: always() && needs.acceptance.result != 'skipped'
      steps:
        - uses: actions/checkout@v6
        - uses: actions/setup-go@v6
          with:
            go-version-file: 'go.mod'
            cache: true
        - run: go mod download
        - run: make sweep
          env:
            VASTAI_API_KEY: ${{ secrets.VASTAI_API_KEY }}
    ```

    Key details:
    - `if: always() && needs.acceptance.result != 'skipped'` ensures sweeper runs even if acceptance tests fail (which is the main reason for leaks)
    - Only runs on main branch (same gate as acceptance tests)
    - 15-minute timeout is generous for cleanup operations
    - NOTE: GitHub Actions does not allow two `if` conditions on the same job; merge them into a single condition: `if: always() && github.ref == 'refs/heads/main' && needs.acceptance.result != 'skipped'`
